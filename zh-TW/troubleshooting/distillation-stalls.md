> *若發現人格蒸餾功能跑很久無法完成、卡在某個階段或產出的人格品質明顯不佳，請參考本指南進行原因診斷與排解*

## 1. 確認 AI 後端是否支援網路搜尋

人格蒸餾的 Phase 1 需要進行**六維度的即時網路調研**，是否有可用的搜尋工具直接決定了整個流程能不能跑：

- **建議使用 CLI 後端**(Claude Code、Gemini CLI、Codex CLI)：這些後端內建搜尋工具，開箱可用
- 若選用 API 後端，必須額外配置搜尋工具，否則 Phase 1 會直接失敗或產出空泛內容。

**切換位置**：設定 → AI → Functions → 人格蒸餾

## 2. 卡在 Phase 1 (六維調研) 太久

Phase 1 要在公開網路上做著作 / 對話 / 表達 / 他者 / 決策 / 時間線六個面向的搜尋，正常情況約需 2 至 5 分鐘。若卡超過 10 分鐘：

- **檢查網路**：CLI 後端的搜尋工具需要穩定外網連線
- **檢查 CLI 工具是否已登入授權**：例如 Claude Code 需先`claude login`
- **確認人物公開資料量**：過於冷門或近期才公開亮相的人物可能讓搜尋反覆撲空，AI 會在「誠實邊界」標註資料薄弱

## 3. 產出人格的口氣 / 判斷與本人差異很大

多數情況代表 Phase 1 抓到的資料不足以支撐 Phase 2 的綜合提煉：

- **補上 Wikipedia 連結**：可加快事實校正，並提供穩定的傳記基礎
- **提供代表連結**：最多 3 個此人最具代表性的訪談、演講或著作 URL
- **指定分類**：例如將張忠謀指定為「商業與金融」「領導與管理」，避免 AI 自動推斷錯方向
- 檢視 persona 檔案末尾的**誠實邊界**段落，AI 會誠實標出哪些維度資料不夠

## 4. 本機 Ollama 後端產出品質低落

人格蒸餾對模型的搜尋與推理能力要求都高，本機小模型 (7B–13B) 通常會出現：

- **Phase 1 直接失敗**：本機模型多半沒有搜尋工具串接，或工具呼叫格式不穩定
- **Phase 2 結構崩壞**：心智模型 / 決策啟發法 / 表達 DNA 的格式無法穩定產出
- **Phase 4 驗證大量 Fail**：誠實邊界塞滿弱點。

**處理對策**：人格蒸餾建議優先使用 CLI 後端或雲端高級模型。本機模型可作為日後人格運行時的對話後端，但蒸餾過程仍以雲端為主

## 5. 中途中止後想重跑

蒸餾流程支援隨時中止；中止後不會留下半成品的人格檔案 (人格檔在 Phase 3 才寫入)。重新觸發即可，可考慮：

- 從**歷史紀錄**找回上次的輸入參數，避免重打
- 調整 Wikipedia 連結或代表連結，可能讓重跑後的調研更穩定

## 6. 持續性失敗的診斷方式

若同一個人物反覆失敗：

- 切換 AI 後端再試一次 (例如從 Gemini CLI 切到 Claude Code)
- 嘗試**更具國際辨識度的人物**做最小化測試 (例如 Steve Jobs)，確認後端與搜尋工具是否正常
- 若問題依舊，請開啟 GitHub Issue 並檢附當下的**輸入參數**與 Console 的完整錯誤訊息

> **ℹ️ Info**
> 人格蒸餾屬於**重量級運作**，視後端搜尋速度與人物資料豐富度，通常需耗時 5 至 10 分鐘。蒸餾程序會在背景執行，可以繼續進行筆記撰寫；任務完成後系統會跳出 Toast 通知
