> *若发现人格蒸馏功能跑很久无法完成、卡在某个阶段或产出的人格品质明显不佳，请参考本指南进行原因诊断与排解*

## 1. 确认 AI 后端是否支援网络搜索

人格蒸馏的 Phase 1 需要进行**六维度的即时网络调研**，是否有可用的搜索工具直接决定了整个流程能不能跑：

- **建议使用 CLI 后端**(Claude Code、Gemini CLI、Codex CLI)：这些后端内建搜索工具，开箱可用
- 若选用 API 后端，必须额外配置搜索工具，否则 Phase 1 会直接失败或产出空泛内容。

**切换位置**：设置 → AI → Functions → 人格蒸馏

## 2. 卡在 Phase 1 (六维调研) 太久

Phase 1 要在公开网络上做著作 / 对话 / 表达 / 他者 / 决策 / 时间线六个面向的搜索，正常情况约需 2 至 5 分钟。若卡超过 10 分钟：

- **检查网络**：CLI 后端的搜索工具需要稳定外网连线
- **检查 CLI 工具是否已登录授权**：例如 Claude Code 需先`claude login`
- **确认人物公开资料量**：过于冷门或近期才公开亮相的人物可能让搜索反复扑空，AI 会在「诚实边界」标注资料薄弱

## 3. 产出人格的口气 / 判断与本人差异很大

多数情况代表 Phase 1 抓到的资料不足以支撑 Phase 2 的综合提炼：

- **补上 Wikipedia 链接**：可加快事实校正，并提供稳定的传记基础
- **提供代表链接**：最多 3 个此人最具代表性的访谈、演讲或著作 URL
- **指定分类**：例如将张忠谋指定为「商业与金融」「领导与管理」，避免 AI 自动推断错方向
- 检视 persona 文件末尾的**诚实边界**段落，AI 会诚实标出哪些维度资料不够

## 4. 本机 Ollama 后端产出品质低落

人格蒸馏对模型的搜索与推理能力要求都高，本机小模型 (7B–13B) 通常会出现：

- **Phase 1 直接失败**：本机模型多半没有搜索工具串接，或工具呼叫格式不稳定
- **Phase 2 结构崩坏**：心智模型 / 决策启发法 / 表达 DNA 的格式无法稳定产出
- **Phase 4 验证大量 Fail**：诚实边界塞满弱点。

**处理对策**：人格蒸馏建议优先使用 CLI 后端或云端高级模型。本机模型可作为日后人格运行时的对话后端，但蒸馏过程仍以云端为主

## 5. 中途中止后想重跑

蒸馏流程支援随时中止；中止后不会留下半成品的人格文件 (人格档在 Phase 3 才写入)。重新触发即可，可考虑：

- 从**历史纪录**找回上次的输入参数，避免重打
- 调整 Wikipedia 链接或代表链接，可能让重跑后的调研更稳定

## 6. 持续性失败的诊断方式

若同一个人物反复失败：

- 切换 AI 后端再试一次 (例如从 Gemini CLI 切到 Claude Code)
- 尝试**更具国际辨识度的人物**做最小化测试 (例如 Steve Jobs)，确认后端与搜索工具是否正常
- 若问题依旧，请打开 GitHub Issue 并检附当下的**输入参数**与 Console 的完整错误消息

> **ℹ️ Info**
> 人格蒸馏属于**重量级运作**，视后端搜索速度与人物资料丰富度，通常需耗时 5 至 10 分钟。蒸馏程序会在背景执行，可以继续进行笔记撰写；任务完成后系统会跳出 Toast 通知
